# JD 商品提取 — 用户使用指南

本扩展用于在京东网站上导出商品数据，保存为 `.jsonl` 文件。数据分两类，**互不混合**：

| 类型 | 说明 | 典型文件名 |
|------|------|------------|
| **详情 JSONL** | 商品详情页的完整信息（标题、规格、变体、长描述等） | `jd-details-2026-05-26-1234567890.jsonl` |
| **链接 JSONL** | 搜索页收集的商品链接（URL、SKU、标题、关键词等） | `jd-links-2026-05-26-手机.jsonl` |

弹窗底部会显示：`当前缓存：详情 239 条，链接 29 条`（数字仅为示例）。

---

## 一、安装扩展

1. 获取本项目的文件夹（需包含 `manifest.json`）。
2. 打开浏览器扩展页面：
   - **Chrome**：`chrome://extensions`
   - **Edge**：`edge://extensions`
3. 打开 **「开发者模式」** → **「加载已解压的扩展程序」** → 选中项目文件夹。
4. 将扩展图标固定到工具栏。

> 无需安装 Node.js，安装后直接在京东网页使用即可。

---

## 二、能用在哪些页面？

| 页面 | 能做什么 |
|------|----------|
| 商品详情 `item.jd.com/xxx.html` | 提取完整商品信息 → **详情缓存** |
| 搜索/列表 `search.jd.com`、`list.jd.com` | 缓存链接 → **链接缓存**；再批量或逐条抓详情 → **详情缓存** |

---

## 三、商品详情页

1. 打开京东商品页，等待加载完成。
2. 点击扩展图标 **「提取并下载 JSONL」**，或页面右下角同名按钮。
3. 扩展会自动滚屏后提取，并下载 **详情 JSONL**。

| 按钮 | 作用 |
|------|------|
| **只提取商品信息** | 只写入详情缓存，不下载文件 |
| **复制 JSON** | 复制最近一次提取结果 |
| **下载详情 JSONL** | 导出详情缓存中的全部记录 |
| **清空详情缓存** | 删除全部详情记录（不可恢复） |

---

## 四、搜索页：三个核心功能

在搜索页打开扩展后，可设置 **自动翻页页数**（`1` = 仅当前页）和 **详情页间隔**（建议 ≥ 2500 毫秒）。

页面右下角浮动按钮与弹窗功能一致：

### 1. 翻页缓存链接

- **做什么**：在搜索结果里自动滚屏、翻页，把每个商品的 **链接**（URL、SKU、标题、搜索关键词等）存进 **链接缓存**。
- **不做什么**：不会打开商品详情，也不会写入详情缓存。
- **之后**：可点「下载链接 JSONL」导出，或继续下一步批量抓详情。

### 2. 批量详情提取

- **做什么**：按 **链接缓存** 里的地址，逐个在后台打开商品详情页 → 滚屏 → 提取完整信息 → 写入 **详情缓存**。
- **前提**：链接缓存不能为空（需先执行「翻页缓存链接」）。
- **完成后**：通常会下载 **详情 JSONL**（`jd-details-*.jsonl`）。

### 3. 逐一点开详情提取

- **做什么**：不依赖链接缓存；在当前搜索页（可多页）上，**逐个模拟点击**每个商品，进入详情提取完整信息。
- **适合**：想一步完成「搜索 → 全部详情」，不想分两步。
- **注意**：保持搜索页标签在前台；商品多时要等很久。多页结束时可选自动下载详情 JSONL。

### 推荐流程（分两步，更可控）

```
搜索页 → 翻页缓存链接 → （可选）下载链接 JSONL 备份
      → 批量详情提取 → 下载详情 JSONL
```

或一步完成：

```
搜索页 → 翻页逐一点开详情提取 → 等待完成 → 下载详情 JSONL
```

### 搜索页弹窗按钮

| 按钮 | 作用 |
|------|------|
| **翻页缓存链接** | 同浮动按钮「翻页缓存链接」 |
| **批量详情提取** | 同浮动按钮「批量详情提取」 |
| **逐一点开详情提取** | 按「自动翻页页数」决定抓当前页或多页 |
| **下载链接 JSONL** | 仅导出链接缓存 |
| **下载详情 JSONL** | 仅导出详情缓存 |
| **清空链接缓存** | 只删链接，不影响详情 |
| **清空详情缓存** | 只删详情，不影响链接 |

---

## 五、两类 JSONL 的区别

**详情 JSONL** 每行是一条完整商品（含 `description`、`variants`、`specifications` 等）。

**链接 JSONL** 每行是一条链接记录，例如：

```json
{"url":"https://item.jd.com/1234567890.html","sku":"1234567890","title":"…","search_keyword":"手机","search_page":"1","collected_at":"…"}
```

## 六、常见问题

**状态栏显示「详情 239 条，链接 29 条」是什么意思？**  
→ 两类数据分开存放；分别用「下载详情 JSONL」和「下载链接 JSONL」导出。

**以前用过「列表提取」，详情里混进了只有标题价格的简版数据？**  
→ 新版本已不再把搜索列表写入详情缓存。若旧数据混杂，可 **清空详情缓存** 后重新批量提取。

**批量详情提示链接缓存为空**  
→ 先在搜索页点 **翻页缓存链接**。

**提取超时**  
→ 减少翻页页数、增大详情间隔；商品页保持在前台。

**重装扩展后数据没了**  
→ 请及时分别下载两类 JSONL 备份。

## 七、快速对照

| 目标 | 操作 |
|------|------|
| 导 1 个商品完整信息 | 商品页 → 提取并下载 JSONL |
| 只要商品链接列表 | 搜索页 → 翻页缓存链接 → 下载链接 JSONL |
| 要一批完整详情（两步） | 翻页缓存链接 → 批量详情提取 |
| 要一批完整详情（一步） | 翻页逐一点开详情提取 |
